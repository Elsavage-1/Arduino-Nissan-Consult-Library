About it

This is the home page of the empeg-consult program, written by Patrick Cole.

Nissan Consult is a serial interface to nissan ECUs that lets you monitor sensors, do active tests, and much more. It is essentially a form of On Board Diagnostics for Nissan manufactured cars.

The purpose of this program is to allow you to use a consult interface with your empeg, having connected it to the serial port of your docking sled.

At present, the program successfully monitors 10 sensors on the screen at once in real time (this seems to be about as much as I can cram onto the empeg display). No active tests or data logging are implemented yet.

If you do not have a consult interface, PL&MS developments can build you a fully working interface ready to go, all you need in addition is a null modem adapter for the sled. Optionally, you can build the interface yourself from the schematic, but be warned I managed to build mine fine on the bread board, but I fried some stuff building the PCB, as some of the components are small and seldom easy to solder.

I will be adding some more functionality (ie. active tests, and background monitoring with audio alert) when I get some time to fiddle.

    Installation

    Download the empeg-consult binary above.

    FTP to your player and install it in /bin.

    Make sure you have "Serial Port Assignment" set to "Apps Use Serial Port".

    The sled serial port connector have RX/TX crossed over, so you will need a null modem adapter between your interface and the empeg sled's serial port.

    Make sure your config.ini has car_rate set to 9600 as this is the rate that the consult interface talks at.

    You may or may not need to have the suppress_notify hijack option to stop the player hollaring things to the serial port. I am not sure if "Apps Use Serial Port" stops the player doing this.

    Execute the program (/bin/empeg-consult) through telnet or at startup using hijack @EXEC features (DO NOT USE THE SERIAL CONSOLE TO START THE PROGRAM).

    Test it by locating the "Empeg Consult" menu item in the hijack menu and launching it. The program will attempt to contact the ECU and will tell you if it does not work.

    Known Issues

    None at this time.
